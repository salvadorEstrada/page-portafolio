Relational database model

Data model exams

This is the database relationship model from which the project was carried out. This model involves three tables, Category, Exam, and Questions. So there are One-to-Many relationships between the Category and the Exam table. On the other side is the same concept for the Exam and Question table. This means that one row in the table Categories links to many rows in the table Exam and that is the same relationship between Exam and Question table.

Data model users

This is the database relationship model regarding users which means that multiple records in the User table are associated with multiple records in the Role table.
On the other side user-rol table is an intermediate table that allows two tables to act as though a relationship were one-to-many for both
